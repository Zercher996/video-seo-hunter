## 项目简介

Video SEO Hunter 是一款 YouTube 视频 SEO 关键词分析工具。它帮助视频创作者分析竞品视频标题，拆解关键词组合，并自动搜索 YouTube 上同类高流量视频，从而发现最有潜力的 SEO 关键词。

**核心价值**: 输入一个视频标题 → 拆词 → 自由组合关键词 → 一键搜索 YouTube → 找到同赛道流量最高的视频。

## 环境准备

### 1. 安装 Node.js

需要 Node.js 18 或以上版本。前往 https://nodejs.org/ 下载安装。

验证安装：

```bash
node -v   # 应输出 v18.x.x 或更高
npm -v    # 应输出 9.x.x 或更高
```

### 2. 获取 YouTube API Key

1. 访问 [Google Cloud Console](https://console.cloud.google.com/)
2. 创建一个新项目（或选择已有项目）
3. 在「API 和服务」→「库」中搜索并启用 **YouTube Data API v3**
4. 在「API 和服务」→「凭据」中点击「创建凭据」→ 选择「API 密钥」
5. 复制生成的 API Key

### 3. 配置环境变量

在项目根目录下创建 `.env` 文件（可参考 `.env.example`）：

```env
VITE_YOUTUBE_API_KEY=你的YouTube_API_Key
```

## 使用流程

### Step 1: 输入视频标题

在输入框中粘贴你想分析的视频标题（英文），然后点击 **Split Words** 按钮（或按 Enter 键）。

> 示例: `AC 85V-250V Multimode Learning Wireless Remote Control Switch 1CH Relay Receiver Module`

### Step 2: 查看分词结果

系统会自动将标题按空格拆分成独立的词语，每个词以彩色气泡的形式展示。

> 上面的示例会被拆分为: `[AC]` `[85V-250V]` `[Multimode]` `[Learning]` `[Wireless]` `[Remote]` `[Control]` `[Switch]` `[1CH]` `[Relay]` `[Receiver]` `[Module]`

### Step 3: 组合关键词

点击你想组合的词语气泡（被选中的气泡会高亮放大），然后：

- 点击 **+ Add Keyword** 将组合添加为一个搜索关键词
- 点击 **Clear** 清除当前选择重新选

你可以反复操作，添加多个不同的关键词组合。

> 示例组合:
> - 选中 `[Remote]` `[Control]` `[Switch]` → 关键词: `Remote Control Switch`
> - 选中 `[1CH]` `[Relay]` → 关键词: `1CH Relay`
> - 选中 `[Wireless]` `[Receiver]` `[Module]` → 关键词: `Wireless Receiver Module`

### Step 4: 搜索 YouTube

点击 **Search YouTube (N keywords)** 按钮开始搜索。系统会逐个关键词搜索，界面上会显示搜索进度。

### Step 5: 查看结果

搜索完成后，每个关键词下会展示该关键词在 YouTube 上**流量最高的前 6 个视频**（时长 2-20 分钟），包括：

- 视频缩略图
- 视频标题
- 频道名称
- 观看数

点击视频卡片可以直接跳转到 YouTube 观看。


## 搜索策略说明

| 参数 | 值 | 说明 |
|------|------|------|
| 搜索范围 | 50 条 | 每个关键词从 YouTube 获取最多 50 条结果（API 上限） |
| 排序方式 | 按播放量 | YouTube API 按 viewCount 排序返回 |
| 时长过滤 | 2~20 分钟 | 去掉太短的片段和太长的视频，保留中等时长 |
| 展示数量 | 前 6 个 | 过滤和排序后，取播放量最高的 6 个视频展示 |
| 搜索方式 | 逐个关键词 | 多个关键词按顺序搜索，避免并发请求压力 |

## FAQ 常见问题

### 基础使用

**Q: 这个工具是做什么用的？**

A: Video SEO Hunter 帮助 YouTube 视频创作者分析竞品标题中的关键词，找到同赛道中播放量最高的视频，从而优化自己的视频标题和 SEO 策略。

---

**Q: 支持中文标题吗？**

A: 目前分词逻辑是按空格拆分的，所以更适合英文标题。中文标题如果词语之间没有空格，会被当成一个整体，无法拆分。

---

**Q: 如何选择关键词组合？**

A: 点击你想组合的词语气泡（可以选多个），它们会按照在原标题中的顺序自动拼接。选好后点击「+ Add Keyword」添加。你可以添加多个不同的关键词组合来对比搜索结果。

---

### API 相关

**Q: 如何获取 YouTube API Key？**

A: 前往 [Google Cloud Console](https://console.cloud.google.com/)，创建项目 → 启用 YouTube Data API v3 → 创建 API 密钥。免费额度足够日常使用。

---

**Q: 每天可以搜索多少次？API 有没有使用限制？**

A: YouTube Data API v3 有免费配额限制：
- **每日配额**: 10,000 单位/天
- **搜索配额成本**:
  - `search.list` 接口：100 单位/次
  - `videos.list` 接口：1 单位/次
- **本工具每次搜索消耗**: 约 101 单位（1次搜索 + 1次获取视频详情）

**所以每天大约可以搜索 99 次**（10000 ÷ 101 ≈ 99 次关键词搜索）

配额会在太平洋时间每天 0:00 重置。如果配额用完，会收到 "YouTube API 配额已用完，请明天再试" 的提示。

---

**Q: 如何查看 API 配额使用情况？**

A: 登录 [Google Cloud Console](https://console.cloud.google.com/) → 选择你的项目 → 「API 和服务」→「已启用的 API」→ 点击 YouTube Data API v3 → 查看「配额」标签页。

---

**Q: 如何申请更多配额？**

A: YouTube API 的默认免费配额是每天 10,000 单位。如果需要更多配额，可以申请配额扩展：
1. 在 Google Cloud Console 中找到 YouTube Data API
2. 点击「配额」标签页
3. 点击「申请更多配额」
4. 填写申请表说明用途

审批通过后，配额会提升到每天 100,000 单位或更多。

---

**Q: 提示「YouTube API Key 未配置」怎么办？**

A: 检查项目根目录下是否有 `.env` 文件，并确保其中包含：
```env
VITE_YOUTUBE_API_KEY=你的API密钥
```
修改 `.env` 文件后需要重启开发服务器（`npm run dev`）。

### 搜索结果

**Q: 为什么某个关键词搜索不到视频（显示 0 videos）？**

A: 可能的原因：
1. **YouTube 上确实没有相关视频** — 关键词太偏或太具体（如某些产品型号）
2. **时长过滤** — 相关视频的时长不在 2~20 分钟范围内（太短或太长都会被过滤）
3. **API 配额用完** — 查看是否有错误提示

---

**Q: 为什么搜索结果和我在 YouTube 搜到的不一样？**

A: 有以下几个原因：
1. **排序方式不同** — 本工具按「播放量」排序，YouTube 默认按「相关度」排序
2. **时长过滤** — 本工具只展示 2~20 分钟的视频，YouTube 默认展示所有时长
3. **结果数量** — 本工具展示前 6 个，YouTube 会展示更多
4. **个性化推荐** — YouTube 网页会根据你的观看历史个性化排序，API 不会

---

**Q: 视频的播放量数据是实时的吗？**

A: 是的，每次搜索都会通过 YouTube API 获取最新的播放量数据。

---

**Q: 为什么有些关键词只返回 1-2 个视频，不到 6 个？**

A: 因为 2~20 分钟的时长过滤。虽然 YouTube 返回了 50 条结果，但过滤后符合时长条件的可能不足 6 个。这对于小众关键词比较常见。

### 部署相关

**Q: 如何部署到线上？**

A: 执行 `npm run build` 生成 `dist/` 文件夹，然后将其部署到任意静态托管服务（如 Vercel、Netlify、GitHub Pages）。注意在托管平台上配置环境变量 `VITE_YOUTUBE_API_KEY`。

---

**Q: 需要后端服务器吗？**

A: 不需要。本项目是纯前端应用，直接在浏览器中调用 YouTube API，无需后端服务器或数据库。

---

**Q: API Key 放在前端安全吗？**

A: 在生产环境中，建议在 Google Cloud Console 中限制 API Key 的使用范围（如限制 HTTP Referrer），防止被滥用。如果对安全性有更高要求，可以搭建一个简单的后端代理来转发 API 请求。
